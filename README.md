# SaaS 功能改版 A/B 实验诊断工具

## 一、项目简介

本项目是一个面向产品团队和数据分析场景的 A/B 实验诊断工具。

用户可以上传功能改版相关的 Excel 数据，系统会自动识别字段、清洗数据、计算核心指标、进行显著性检验，并给出是否建议上线的业务判断。

项目重点不是简单展示图表，而是围绕一个真实业务问题展开：

> 一个功能改版之后，到底有没有带来真实有效的提升？是否值得继续灰度、扩大上线，还是应该暂缓上线？

\---

## 二、项目目标

本项目主要解决以下问题：

1. 改版后用户行为是否变好？
2. 新功能是否提高了点击率、使用率和任务完成率？
3. 改版后是否提高了付费转化率？
4. 改版是否影响 ARPU？
5. 当前样本量是否足够支撑上线判断？
6. 指标提升是否具有统计显著性？
7. 最终应该上线、继续灰度，还是暂不上线？

\---

## 三、核心功能

### 1\. Excel 数据上传

系统支持两种上传方式：

#### 方式一：单个 Excel 文件

支持以下三种格式：

1. **单 sheet 用户级宽表**  
一行一个用户，包含用户 ID、分组、点击、使用、完成、付费、金额等字段。
2. **双 sheet 前后对比表**  
一个 sheet 表示改动前 / before，另一个 sheet 表示改动后 / after。
3. **多 sheet 标准实验数据**  
包含 users、assignment、events、payments 等工作表。

#### 方式二：两个 Excel 文件

用户可以分别上传：

* 改动前数据
* 改动后数据

系统会自动将：

* 改动前数据识别为 control 组
* 改动后数据识别为 treatment 组

这种方式适合真实业务中常见的两个导出文件场景。

\---

### 2\. 字段自动识别

系统可以识别常见字段同义词。

|标准字段|可识别字段示例|
|-|-|
|user\_id|user\_id、userID、用户ID、用户名、账号、客户ID|
|group|group、实验组、分组、版本、AB组|
|period|period、阶段、改版前后、上线前后|
|click\_feature|点击、功能点击、新功能点击、是否点击|
|use\_feature|使用、功能使用、新功能使用、是否使用|
|complete\_task|任务完成、完成任务、文档完成、完成流程|
|is\_paid|是否付费、付费、是否转化、购买|
|amount|金额、消费金额、订单金额、收入、GMV、实付金额|

系统会尽量将不同命名习惯的数据字段转换为统一分析口径。

\---

### 3\. 数据清洗

项目会对上传数据进行基础清洗，包括：

* 去除重复行
* 统一 user\_id 格式
* 处理缺失值
* 统一日期格式
* 过滤非法实验分组
* 修正异常金额
* 检查用户 ID 是否能匹配
* 将用户行为数据转换为用户级指标

\---

### 4\. 核心指标计算

系统会按 control 组和 treatment 组分别计算：

|指标|含义|
|-|-|
|新功能点击率|用户是否点击过新功能|
|功能使用率|用户是否实际使用过功能|
|任务完成率|用户是否完成关键任务|
|付费转化率|用户是否产生付费行为|
|ARPU|每个用户平均收入|
|Uplift|treatment 相比 control 的提升幅度|

\---

### 5\. A/B Test 显著性检验

系统根据指标类型使用不同检验方法：

#### 比例类指标

使用 **Proportion Z-Test**，适用于：

* 点击率
* 功能使用率
* 任务完成率
* 付费转化率

#### 收入类指标

使用 **Welch T-Test**，适用于：

* ARPU
* 用户平均收入

默认显著性水平为：

```text
alpha = 0.05
```

当：

```text
p-value < alpha
```

系统认为两组差异具有统计显著性。

\---

### 6\. 样本量判断规则

系统不会只根据 p-value 给结论，还会结合样本量判断结论力度。

|每组样本量|判断方式|
|-|-|
|< 50|样本量过小，不建议做上线判断|
|50 - 100|只能作为方向性参考|
|100 - 300|可做初步判断，适合继续灰度|
|300 - 1000|可以给出较稳的灰度上线建议|
|≥ 1000|若核心指标显著且无负面影响，可建议扩大上线|

这样可以避免小样本下过度解读实验结果。

\---

## 四、页面功能

上传数据后，页面会展示：

1. **实验结论**  
根据核心指标、显著性检验和样本量规则生成上线建议。
2. **核心指标**  
展示功能使用率、任务完成率、付费转化率、ARPU 等关键指标。
3. **可视化分析**  
对 control 和 treatment 的核心指标进行对比展示。
4. **A/B Test 显著性检验**  
展示 p-value、置信区间、uplift 和显著性结论。

## 五、支持的数据格式

### 1\. 单 sheet 用户级宽表

示例字段：

|用户ID|分组|新功能点击|新功能使用|任务完成|是否付费|消费金额|
|-|-|-|-|-|-|-|
|U001|control|1|0|0|0|0|
|U002|treatment|1|1|1|1|99|

适合已经整理好的一张实验明细表。

\---

### 2\. 双 sheet before / after

一个 Excel 文件中包含两个工作表：

```text
before / 改动前
after / 改动后
```

系统会自动将：

```text
before → control
after → treatment
```

适合分析功能改版前后的效果变化。

\---

### 3\. 多 sheet 标准实验数据

一个 Excel 文件中包含：

```text
users
assignment
events
payments
```

|sheet|含义|
|-|-|
|users|用户基础信息|
|assignment|实验分组信息|
|events|用户行为事件|
|payments|用户付费数据|

适合结构较完整的业务数据。

\---

### 4\. 双文件上传

用户也可以分别上传：

```text
改动前数据.xlsx
改动后数据.xlsx
```

系统会自动将：

```text
改动前文件 → control
改动后文件 → treatment
```

适合真实业务中常见的两个导出文件场景。

## 七、项目结构

```text

│
```

|文件|作用|

## 八、如何运行

### 1\. 安装依赖

```bash
pip install -r requirements.txt
```

### 2\. 启动项目

```bash
streamlit run app.py
```

启动后，在浏览器中打开 Streamlit 页面即可使用。

\---

## 九、使用方式

### 方式一：上传真实 Excel 数据

1. 在左侧选择“上传 Excel 数据”
2. 选择上传方式：

   * 单个 Excel 文件
   * 两个 Excel 文件：改动前 + 改动后
3. 上传文件
4. 系统自动识别字段并完成分析
5. 查看上线建议、核心指标、显著性检验和图表结果

### 方式二：加载示例数据

如果暂时没有真实数据，可以选择“加载示例数据”体验完整流程。

注意：示例数据仅用于演示系统功能，不代表真实业务结论。
